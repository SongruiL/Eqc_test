# EQC 优化层设计规格（spec-optimization）

> 状态：**设计已定稿（2026-06-19，与首席科学家逐点讨论确认）**，待分阶段实现。
> 本文是动手前的唯一参照。命名以本文为准。

## 0. 这一层解决什么

前向模型（解释器 `eval`/`sim`）回答的是：**「当前条件下，系统会变成什么样？」**

但一套优秀的决策工具要回答的是：**「想让系统变成某样（产量最大、特定时间上市、成本最低……），我们该怎么做？」**

我们**不解析地反推方程**（机理模型含阈值/分段，无法求逆）。而是**在前向模型上搜索**：试一组决策 → 跑前向模型 → 给结果打分 → 调整再试。这叫**仿真优化（simulation-based optimization）**。

## 1. 三层架构（解释器是共享底座）

```
  ┌────────────────────────────────────────────────────────┐
  │ 搜索算法（可插拔）：网格(=eqc sweep, 已有) / 差分进化 DE /  │  ← 怎么找
  │   CMA-ES / 贝叶斯 / 梯度(future) / GP(进化结构, future)    │
  ├────────────────────────────────────────────────────────┤
  │ 目标评估核：给定旋钮赋值 → 跑前向模型 → 归约成一个标量目标  │  ← 评价一个候选
  │   （决策优化 / 参数标定 / GP-fitness 全部共用这一层）       │
  ├────────────────────────────────────────────────────────┤
  │ 前向模型 + 解释器（src/sim, src/eval）：f(输入)→逐日轨迹    │  ← 算一次（唯一权威）
  └────────────────────────────────────────────────────────┘
```

**关键统一**：决策优化、参数标定、GP（结构进化）在架构上是**同一个外循环**——「跑模型 → 打分 → 搜索」。区别只在两点：

| 用途 | 调的「旋钮」 | 「分数」（目标） |
|------|-------------|------------------|
| **决策优化** | 可控管理变量（温度/CO₂/播种期/水肥…） | 产量 / 利润 / 上市时间（要最大化/命中的目标）|
| **参数标定** | 模型参数 | 预测 vs 实测的误差（最小化）|
| **GP / 结构学习** | 模型结构（S 表达式基因型）| 误差 + 简约性 + 约束惩罚（= fitness）|

所以**目标评估核只写一次，三种用途全复用**；解释器是它们共同的底座。`eqc sweep` 已是这套的最简版（1 维网格优化器 + 单变量归约）。

## 2. 旋钮（决策变量）怎么设计——核心原则

> **「可控」不是变量的固有属性，而是『问题/场景』的属性 → 不写进模型，放在模型之上单独一层（决策 spec）。** 与「模型结构 vs 情景数据」分离同一个哲学。

同一变量在不同问题里角色不同：温室温度优化运行策略时是决策、优化品种时是固定环境；CO₂ 在温室可控、露天不可控；上界/成本是设备与经济事实，模型不知道。所以「可控性」必须可按问题切换 → 独立声明。

### 2.1 「可控」的两个条件（对应「看方程」+「看现实」）

| 条件 | 谁判断 | EQC 如何支持 |
|------|--------|--------------|
| **必要：模型得响应它** | 必须是模型的**外部输入**（参数 / 初值 / 驱动），不是被方程算出的中间量/状态 | **EQC 自动识别候选旋钮** = 所有外部输入（`eqc export` 契约已含；情景面板已在列）。再用 `eqc sweep --sensitivity` 筛掉「调了没影响」的 |
| **充分：现实能设、设到多少、什么代价** | 设备/成本/农艺约束 | EQC 不可能知道 → 由人在决策 spec 里声明（边界、成本、离散档位）|

这把「判断可控变量」这件高要求的事，切成 **EQC 能自动给的部分（结构 + 敏感性）** 和 **必须人填的部分（现实可行性）**。这套抽象**领域无关**：温室设计、土壤治理、化工反应器、任何机理模型都有参数/初值/驱动，都能配一份决策 spec——因为我们没引入任何领域概念，只引入了「外部输入 + 边界 + 目标表达式」三个数学概念。

### 2.2 旋钮种类（标量先行，留好推广口子）

旋钮统一抽象为「**一个数（或几个数）→ 赋给某个外部输入 / 参数化某条输入曲线**」。

1. **设计变量**（设一次、不随时间）：种植密度、温室高度、品种参数 → **标量旋钮**。✅ 第一版
2. **恒定控制**（一个设定值）：固定 CO₂ / 固定设定温度 → **标量旋钮**（赋给一个 driver 的恒定值）。✅ 第一版
3. **时变控制**（逐日曲线）：每天的温度/CO₂ 设定 = 最优控制，维度爆炸。务实做法：**用少量旋钮参数化输入曲线**（如 温度 = 基线 + 振幅·季节型，只优化 2 个数）。🔜 抽象允许，后置
4. **离散/分类**（品种、开关、作物种类）：DE 连续，离散需取整/混合整数。🔜 后置

## 3. 目标 / 约束 = S 表达式（领域无关、可检视、复用解释器）

**「目标也是一条方程」**：目标和约束都写成 S 表达式，交给解释器算。好处：复用解析器+AST+解释器、领域无关、可检视、自己吃自己的狗粮。

### 3.1 目标活在「整季归约」空间

模型方程是**逐日**算的；目标是给**整段轨迹**打分。所以目标方程的「变量」是对轨迹的**时间归约**，需要一小套**时间归约词汇**：

- `final(X)` —— 末值（最常用：最终产量）
- `at(X, t)` —— 第 t 天的值（命中上市窗口）
- `max(X)` / `min(X)` / `mean(X)` / `total(X)` —— 峰值 / 谷值 / 均值 / 全季累计
- 🔜 后置：`argmax_day(X)`（峰值出现在第几天，做「时间」类目标）

> **与 `vsum` 不同**：`vsum` 是「一步之内对向量分量求和」；时间归约是「跨天对一条序列归约」。两者不可混。
> 时间归约是 `eqc sweep --reduce final|max|mean|min` 的**推广**：sweep 是单变量单归约，目标方程是多变量、归约 + 任意算术组合。

目标/约束方程除轨迹归约外，还能引用**旋钮值与参数**（成本项要用）。例：

```
# 利润最大化：末产量·单价 − CO₂ 成本
max  (sub (mul (final Y) price) (mul CO2_setpoint co2_cost))

# 命中上市期：让第 120 天产量贴近目标
min  (abs (sub (at Y 120) target_yield))
```

### 3.2 约束

约束也是 S 表达式（对轨迹归约 / 旋钮）：
- **箱形边界**（每个旋钮 `[lo, hi]`）：DE 原生支持。
- **耦合/一般约束**（如能耗、预算 `expr ≤ 0`）：用**惩罚法**（违反时给目标加大额惩罚）。

## 4. 决策 spec 格式（与模型文件分离的独立产物）

```yaml
optimize:
  objective:
    expr: (sub (mul (final Y) price) (mul CO2_setpoint co2_cost))  # 目标 S 表达式
    sense: max                                                      # max / min
  knobs:
    - { var: CO2_setpoint, kind: driver_const, bounds: [400, 1200], unit: ppm }  # 恒定驱动
    - { var: Pd,           kind: param,        bounds: [5, 10] }                  # 标量参数
    - { var: TF0,          kind: init,         bounds: [0, 100] }                 # 状态初值
  constants:                       # 目标/约束里用到的非模型量（单价、成本系数、目标值…）
    price: 30.0
    co2_cost: 0.002
  constraints:
    - { expr: (sub (total energy) budget), max: 0 }   # 总能耗 ≤ 预算
  environment: weather.csv          # 未被选为旋钮的驱动 = 不可控环境
  optimizer:
    method: de                      # 第一版：差分进化
    pop: 30
    iters: 100
    seed: 42                        # 确定性：定种子 → 结果可复现
```

> 注：`constants` 让目标/约束方程引用模型之外的经济/目标量；`environment` 提供不可控驱动的默认时间序列。

## 5. 优化器：从 DE 起步

- **差分进化 DE**（derivative-free）：把模型当黑盒、只需能评估目标。对作物模型常见的**非光滑 / 阈值 / 分段 / 多峰**鲁棒；契合首席科学家原意。**第一版唯一优化器。**
- **为何不上梯度下降**：模型满是阈值/门控（物候、`geq`），目标对决策非光滑甚至不连续，梯度不可靠；自动微分是大投入且要模型光滑 → 后置。
- **确定性**：DE 用随机数 → 优化器**接受种子**，定种子结果可复现（与项目「输出可复现」一致）。
- **垃圾候选鲁棒**：某些旋钮组合会让 sim 出 NaN/发散 → 评估核**捕获并赋最差目标值**（惩罚），不崩溃（呼应「GP 时让 NaN 当惩罚」）。

## 6. CLI（拟）

```
eqc optimize <模型> --spec problem.yaml [-o result.json]
# 读 模型 + 决策 spec；DE 搜旋钮空间；输出最优旋钮 + 目标值（+ 收敛轨迹）
```

`eqc sweep`（已有）= 单旋钮网格扫描 + `--sensitivity` 全局敏感性，作为优化前的**预筛**（哪些旋钮值得纳入搜索）。

## 7. 复用与依赖关系

- **目标评估核** = `(模型, 旋钮赋值) → 跑 sim → 用 S 表达式目标 + 时间归约归约成标量（+ 约束惩罚）`。决策优化 / 标定 / GP-fitness **全共用**。
- **标定**只是「旋钮=参数、目标=误差vs实测」；**GP** 再往上把「旋钮」换成整套结构。
- **顺序依赖（重要）**：在**未标定**的模型上做决策优化 = 拿错模型推错决策。真正解锁链：**田间数据 → 标定/进化出可信模型 → 在可信模型上优化决策**。GP/标定 与 决策优化不是二选一，而是先后，且共用同核。

## 8. 分阶段实现计划

- **阶段 1（第一步，本设计的落地目标）**：目标评估核 + 决策 spec（**仅标量旋钮**：param / init / 恒定 driver）+ **DE 优化器** + 时间归约词汇（final/at/max/min/mean/total）+ `eqc optimize --spec`。验证：在草莓 S4 上「扫 CO₂/Pd 找产量或利润最优」，与手算/`eqc sweep` 网格结果一致。
- **阶段 2**：约束（惩罚）、多输出/多目标雏形、`--sensitivity` 自动预筛接入优化、`eqc serve`/Studio 里可视化收敛与最优。
- **阶段 3+（后置）**：曲线参数化的时变控制、离散旋钮、参数标定（接田间数据）、其它优化器（CMA-ES/贝叶斯）。
- **更远**：GP 约束进化层（复用目标评估核当 fitness 引擎；需田间数据 + 先定可进化/冻结节点与约束）。

## 9. 与现有代码的衔接

- 候选旋钮来自 `src/export.rs` 契约（参数/变量 + 类/单位/是否向量）；向量参数（cohort 种子）不可做标量旋钮（沿用情景面板的跳过逻辑）。
- 目标/约束 S 表达式复用 `ast`（解析）+ `eval`（求值）；时间归约是**新增的一小套作用于 `SimOutput` 的归约**（不是逐日算子、不是 `vsum`）。
- 前向运行复用 `sim::simulate` + `sim::build_plan`。
- `eqc sweep` 的 `parse_range`/`reduce_series` 思路可复用/推广。
