# EQC 参数标定层设计规格（spec-calibration）

> 状态：**设计稿（2026-06-20，与首席科学家讨论后）**。云南栽培实验 2026-07 起、3–4 个生长周期修正模型 → 标定工具需在数据到来前备好、并能跨周期复用。
> 本文是动手前的参照；命名以本文为准。配套：`docs/spec-optimization.md`（标定复用其评估核）。

## 0. 这一层解决什么

机理模型的**结构**（方程）来自文献，但**参数**多为估计/借用（如草莓 S4 的 `Kc=340` 注明「待重标定」）。**标定 = 用实测数据反推出最吻合现实的参数值**，让模型不仅结构对、量级也对。这是「在未标定模型上做决策 = 拿错模型推错决策」的解药，也是通往 GP 的桥。

## 1. 与优化层同一个外循环（复用评估核）

标定与决策优化是**同一个外循环**「跑模型 → 打分 → 搜索」，只换两处：

| | 旋钮（调什么）| 目标（打什么分）|
|------|------|------|
| 决策优化 | 管理量（CO₂/密度…）| 产量/利润（max）|
| **标定** | **模型参数**（Kc/LUE…）| **预测 vs 实测的误差**（min）|

所以**评估核（`src/optimize/core.rs`）、DE、决策 spec 格式全部复用**。标定 = 「旋钮 kind=param、目标=误差表达式、外加一份实测数据」。

## 2. 唯一的新功能：目标能引用实测数据

现有目标只能归约**仿真**轨迹（`final/at/max/min/mean/total`）+ 常量。标定要写「误差」，需要新增**误差算子**——把**仿真序列**与**实测序列**逐（观测）日比对、归约成一个标量：

- `(rmse SimVar ObsName)` —— 均方根误差 √(mean((sim−obs)²))（最小化）
- `(mae  SimVar ObsName)` —— 平均绝对误差 mean(|sim−obs|)（最小化）
- `(nse  SimVar ObsName)` —— 纳什效率 1 − Σ(sim−obs)²/Σ(obs−mean_obs)²（**最大化**，1=完美）
- `(bias SimVar ObsName)` —— 平均偏差 mean(sim−obs)（配 `abs` 最小化）

实现沿用时间归约的套路：在 **SExpr 层**把误差子式就地替换成一个数（从 `SimOutput[SimVar]` + 实测 `ObsName` 算出），再 `convert`+`eval`。**不新增 AST 变体**。多观测变量可组合，如 `(add (rmse Y obs_Y) (mul w (rmse LAI obs_LAI)))`（加权多目标拟合）。

### 2.1 实测数据格式（稀疏，对应周期性破坏性取样）
实测 CSV：首列 `DAT`（1 起的天数），其余列 = 观测变量。空单元 = 那天没测。内部表示 `ObservedData: 名 → [(天, 值)]`（稀疏）；误差只在**有观测的天**上算（`sim_series[天−1]` 对 `obs`）。

## 3. 标定 spec（复用决策 spec + 一处扩展）

复用 `optimize:` 决策 spec，旋钮 kind=param、目标=误差、外加 `observed:` 指实测 CSV：

```yaml
optimize:
  objective:
    expr: (rmse Y obs_Y)        # 误差（也可多变量加权组合）
    sense: min
  knobs:
    - { var: Kc,  kind: param, bounds: [100, 600] }
    - { var: LUE, kind: param, bounds: [1, 5] }
  observed: field_obs.csv        # 实测数据 CSV（相对 spec 目录）
  environment: weather.csv       # 同期天气驱动（与实测同一季）
  optimizer: { method: de, pop: 30, iters: 100, seed: 42 }
```

> 标定与决策优化共用一套 spec 与一条命令——标定不过是「旋钮=参数、目标=误差、带 observed」的优化。（CLI 形态：复用 `eqc optimize`，或加一个友好别名 `eqc calibrate` + 可辨识性报告——Cal-2 定。）

## 4. 验证：recover-the-params（无需真实数据即可建+验）

1. 选一组「真值」参数，跑一次 sim → 一条轨迹，当**伪实测**。
2. 把这些参数当旋钮、目标 = 与伪实测的 RMSE，从别的初值标定。
3. 看能否把参数**找回到真值**（在可辨识前提下）。

证明标定回路正确、数据一到即可用。

## 5. 可辨识性 / 「该测什么」助手（服务实验设计）

**你只能标定被观测约束住的参数。** 标定前做一次分析，告诉园区「要定准参数 X，最该测变量 Y、在哪些时间点」：

- 对每个候选参数做敏感性（复用 `optimize::prescreen` 思路）：参数扰动对**各候选可观测变量**的影响。
- 报告：哪些参数对哪些观测敏感（可标）、哪些参数无观测能约束（不可辨识，需补测或固定）。
- 警惕**异参同效（equifinality）**：多组参数给出几乎相同输出 → 单一观测下不可辨识。对策：多变量观测 + 处理梯度（不同 CO₂/水肥小区）+ 分阶段标定。

CLI 设想：`eqc identify <模型> --spec calib.yaml`（或 `eqc calibrate --identify`）。

## 6. 科学注意（标定方法论）

- **必须时间序列**：单个期末产量标不出动态速率参数 → 周期性测量。
- **处理梯度**提升可辨识性：不同管理条件「激发」不同参数。
- **分阶段/模块化标定** > 一次标所有：先物候（用开花日）、再生长/分配（用生物量序列）、最后整体微调。
- **报告不确定性**：雏形可先不做完整量化，但要诚实标出哪些参数没被数据约束住。

## 7. 分阶段实现计划

- **Cal-1**：误差算子（`rmse/mae/nse/bias`）+ 实测数据接入目标求值器（库层，不动评估核）。
- **Cal-2**：实测数据贯通评估核（`evaluate` 带 observed）+ 标定 spec（`observed:`）+ CLI（`eqc calibrate` 或复用 `eqc optimize`）。
- **Cal-3**：recover-the-params 合成自验（端到端）。
- **Cal-4**：可辨识性 / 「该测什么」助手，产出云南实验测量清单。
- **（数据到位后）**：真标 S4 / 带根系-水肥的新模型。

## 8. 与模型扩展的关系（根系-水肥等）

标定工具**结构无关**：读模型当前有哪些参数 + spec 列出要标的 + 实测数据 → 工具不变。模型加根系/水肥后：① 新参数自动可标（spec 里列上即可）；② 但要新增**驱动**（灌溉/施肥/含水率）与**能约束新参数的新观测**；③ 参数变多 → 可辨识性更吃紧 → §5 的助手更重要。正因模型会长大，**先把标定做成通用工具**才划算。
